Edit general information about a dictionary (D)

The whole dictionary ›› General information about a dictionary ››

In SooSL Desktop, you can change the general information about a dictionary project.

To do so, first click Tools >   About Dictionary. You will see the "Dictionary Information" dialog box. Then click at the bottom to change the information. You will see the Edit Dictionary Information dialog.

 

You can change basic information about the dictionary, such as 

  • its title (1) 
  • the name of the sign language (2)
  • a version number or some other label for this version of the dictionary (3)

Other information (4) about the dictionary is read-only. SooSL remembers it for you automatically. 

  • Last update: when you last made changes to the dictionary
  • Dictionary folder: the folder where it is stored on your hard disk. (The folder name may be similar to the title of the dictionary, but it doesn't have to be.)
  • Dictionary Project ID: This code is used to distinguish this dictionary project from every other SooSL dictionary project anywhere. When there are different versions of the same project, the Project ID tells SooSL which versions belong to the same dictionary project. It is based on the title of the dictionary and when you first create the dictionary and the time you create it. You cannot change the Project ID and you don't need to.

You can also write a longer general description (5) about the dictionary. In the description, you might want to include information like this:

  • the dictionary's title—usually repeat from (1).
  • the sign language and the written languages in it—usually repeat from (2).
  • the people who worked on it
  • instructions to users
  • people who you want to thank
  • anything else you want your users to know about the dictionary

A simple description can be in plain text. If you want special formatting, you can use HTML codes. When you use HTML, it looks like what you see above, with HTML tags like <h2> and <p> to describe the formatting you want. You can click (6) to see what the description will look like to people using the dictionary. 

Click to go back to editing the HTML code.

Tip: If you use HTML and want to create blank lines, use the <br> tag. Although SooSL will usually interpret a blank line in the HTML code as a blank line in the output, it doesn't always work. The normal way of indicating a blank line in HTML is to use the <br> tag.

When you are done editing, click to keep your changes or click to discard them.
